wirefold — architecture

A concurrent dataflow network where the topology is the logic. Go owns the model; TypeScript only draws it.

One clock, in ticks

There is exactly one clock: the system monotonic clock read through a scale so it advances in integer ticks at human-watchable speed. Halted time is subtracted, so pausing genuinely freezes the tick:

tick = ⌊(now − start − haltedTotal) / tickPeriod⌋  (MsPerTick = 16 → ≈62.5 ticks/sec)

The model is sleep-only. The clock interface is Tick() · SleepCycle(ctx) · Halt() · Resume(). Pacing loops call SleepCycle to wait exactly one cycle rather than blocking on a target tick — there is no WaitTick(k) (the symbol does not exist; only stale comments name it). Halt touches no goroutine — it sets a flag and a timestamp. SleepCycle is a raw timer and is not pause-aware: paused loops keep spinning, they just observe a frozen Tick(). Pause governs beads only — UpdateLayout never waits on the clock and keeps running.

Geometry edits preserve fractional progress

ReviseInFlightGeometry (paced_wire.go) preserves each bead's fraction t along the wire, not its absolute distance covered — so uniform pulse speed still holds after a drag: remaining = (1 − t) · newArc / pulseSpeed. It rebases placementTick for every bead in flight, not just one.

pulseSpeed is an instance field on each wire, so it could differ per wire — but every caller passes the one global constant. Uniform pulse speed is a convention here, not something the type system enforces.

No lockstep rounds, no simultaneity layer. Goroutines schedule independently against the shared tick. Any reasoning that treats activity as globally-aligned rounds is drift.
