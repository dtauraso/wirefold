A concurrent dataflow network where the topology is the logic. Go owns the model; TypeScript only draws it.

Stale self-descriptions found in this audit

Each row is a place the code, or its instructions, assert something untrue. Verified against source on 2026-07-14. Ordered by how likely an agent is to act on it.

CLAUDE.md — the node-label sidecar highest risk

The project's own instruction file describes the bridge as "the binary content buffer plus the node-label id/label/kind sidecar." That sidecar is gone — labels ride the Node block's LABEL_OFF/LABEL_LEN columns into the Label section, and parseHostToWebview.test.ts actively asserts the message type is rejected. MODEL.md agrees with the code. An agent following CLAUDE.md would build against a deleted seam.

CLAUDE.md — the bridge surface omits fade-toggle and mis-states fade highest risk

CLAUDE.md lists the TS→Go surface as "raw-input, spec save/load, and a single geometry-CRUD edit message … plus the play/pause control signal", and says "Fading an edge … [is an] update on their entity." Neither matches the code. fade-toggle is a seventh top-level type and a deliberate bare command: it carries no entity id because Go owns the selection, so TS cannot name an entity to update. The code is right — and enforced (live sender at ThreeView.tsx:99, a test, and fadeToggle:5 inside the guarded INPUT_LAYOUT_FINGERPRINT v10). The doc is what drifted.

stdin_reader.go header — undercounts its own switch confusing, not false

The header enumerates five top-level message types; the switch has seven. Its line that fade edits "no longer cross this seam" appears to mean fade-as-an-edit-op was replaced by the bare command — true, but it reads as a flat contradiction of the fade-toggle case fifteen lines below it.

WaitTick — a symbol that no longer exists

Named in comments in input/node.go, layout_holder.go, and paced_wire.go. Zero definitions repo-wide. The clock has been sleep-only (SleepCycle) for some time. This one propagated straight into the first draft of the Clock page.

"The Center is the ONE cartesian value in the system"

A comment in scene_sphere_persist.go. False as written — camera pose, port anchors, bead segments and derived node centers are all cartesian. The intended invariant (only persisted cartesian anchor; everything else derived or quarantined) is true and worth stating precisely.

Vestigial json:"…" tags on stdinMsg cosmetic, still misleading

Nothing unmarshals them — input_codec.go decodes the binary record. They imply a JSON seam that no longer exists.

overlay-flags.ts references useCameraStore

In a comment only. No such store exists anywhere in src/ — it was erased with the old system. Points an agent at a state-authority pattern the guards exist to forbid.
