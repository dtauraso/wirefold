Trace.mu — the last lock in the repo

Six other mutexes were removed before this one — geomMu and RealClock.mu removed, PacedWire.mu and outbox.mu removed, LayoutHolder.mu and debouncedPersister.mu removed. This one is at Trace/Trace.go:294. Its stated job is guarding the event accumulator — and that turns out to be mostly vestigial. What it actually still does in production is something else.

goroutine under the lock lock-free test-only

1 · the hot path never touches the lock

Every node and wire goroutine emits events. None of them takes the mutex. They send on a buffered channel and return. The lock lives entirely on the far side of that channel, on one goroutine.

3 · who actually takes the lock — and which of them exist in production

This is where the stated justification comes apart. Events() has zero non-test callers, and WriteJSONL runs only after the drain goroutine is already dead.

4 · what the lock is really doing — shutdown, not accumulation

Production shutdown waits at most 250 ms for node goroutines, then calls Close() regardless. So Close can run while goroutines are still calling Breadcrumb. That is genuine production contention — and it is on sink/debugSink, not on events.

5 · unlike every other lock here, this one is held across I/O

Every lock deleted this session released before touching the outside world. This one does not.

6 · the single-owner question — drain already is one

The move that removed the other six: give the state one owner. events already has one — the drain goroutine is its sole writer. So the accumulator was never the obstacle.
