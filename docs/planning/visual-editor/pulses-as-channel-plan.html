Pulses-as-Channel Rebuild Plan

branch: task/pulses-as-instances  — does not ride to main

A Go channel between two nodes is bounded — size 1. At any moment it holds zero or one item. The visual edge is the channel: when an item is in transit it appears as an animated pulse traveling the edge path; when the channel is empty the edge is a plain line.

The animation completing at the destination is the receive completing. Visual pacing drives Go pacing — the receive does not complete until the animation lands. This means the sender blocks for the entire travel duration if it tries to send a second item before the first is consumed.

There is no queue on the wire, no pulse IDs, no dedup refs. The entire state of an edge collapses to a single bit: in-transit or not. On Run restart, channels reset to empty and a new simulation begins. There is no cross-run state to clear.

Today (broken): pulseValueRef and dedup refs in SubstrateEdge.tsx / use-fire-flash.ts carry state across Run restarts. On Run 2, stale ref values from Run 1 suppress the first pulse, so the first item appears to vanish silently.

Deleting these removes every cross-run state assumption:

  • clearRunState function and all call sites
  • run-start event plumbing (pump → webview → edge)
  • runEpoch counter / prop threading
  • pulseValueRef in SubstrateEdge.tsx
  • prev ref in use-fire-flash.ts (dedup guard)
  • Any pulse id concept (pulseId, id field on pulse events)
  • Any pulse queue or array (pulses[], multi-pulse accumulation)
  • Dedup equality checks comparing current vs previous pulse value

Stays:

  • Trace event format from Go — send and receive events still arrive in order
  • Animation CSS / keyframe approach in use-fire-flash.ts (just simplified)
  • Edge component structure in SubstrateEdge.tsx
  • pump.ts event routing loop

Changes:

  • data.pulse field → data.inTransit: boolean (single-slot, no value needed for animation)
  • pump.ts: on send event for an edge → set inTransit = true; on receive event → set inTransit = false
  • SubstrateEdge.tsx: reads data.inTransit, renders pulse when true, plain line when false — no ref, no epoch
  • use-fire-flash.ts: driven by inTransit boolean prop, no dedup logic needed
  • On Run restart: pump resets all edge inTransit to false — no special event required, just a reset pass over the RF store
