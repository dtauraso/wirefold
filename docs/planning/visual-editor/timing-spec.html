Wirefold Timing Spec

branch: task/multi-bead-wire  ·  docs/planning/visual-editor/timing-spec.html

Time in this network is distance ÷ one speed. The single constant pulseSpeed = 0.04 world-units/ms is the only conversion between spatial distance and elapsed time for everything visible in the editor.

time = distance / pulseSpeed distanceCovered = pulseSpeed × Δt
  • Local timing distance. Each node and each wire owns its own local timing distance. Distances are independent; there is no shared global counter beyond the one clock.
  • Uniform human speed. Every distance — wire arc lengths, window widths, dwell, train length, recv-gate — advances at the same pulseSpeed. Nothing travels faster or slower than anything else.
  • One clock, one button. Play/pause is a single control on all timers. Halting the clock freezes all distance accumulation simultaneously; resuming releases them together with no drift.

Node input→output processing is instant today — receive, fire, and send happen in the same tick. This is correct behavior; there is no active plan to add per-node processing delay.
