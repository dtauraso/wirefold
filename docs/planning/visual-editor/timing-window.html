Timing-Window (Coincidence-Detection) Rule

branch: task/timing-window  ·  docs/planning/visual-editor/timing-window.md

A multi-input node should fire only when its required inputs arrive close together in time. Without a timing window, a partial set of inputs accumulates indefinitely — stalling upstream sources and piling up beads at a gate that may never complete.

The timing window opens when the first required input arrives (t0). If all required inputs arrive within window W, the node fires normally. If W elapses with any input still missing, the node clears: it calls Done on every held input (draining upstream wires) without firing, then resets its flags and waits for the next first-arrival.

This directly solves the permanent-delete case: an input edge removed for good means that input will never arrive, so the gate would block forever. With a timing window it times out after W, clears, and lets the ring keep running — no pile-up, no stall.

W is a per-node value stored in node.data (same ownership as send rules). Absent means no window — wait indefinitely, which is today's unchanged behavior for non-windowed nodes.

Top path: both inputs arrive inside W → FIRE (Done all, consume/send). Bottom path: input B never arrives, W elapses → CLEAR (Done drains held input A, no fire, flags reset).

  • Per-node rule in node.data — same ownership as send rules (consumeGated / fireAndForget). Absent ⇒ no window ⇒ blocking Recv, unchanged behavior.
  • First implementation target: inhibitRight0 (the AND-gate that motivated this). readGate stays windowless (ring lockstep requires indefinite wait).
  • Non-blocking poll — add PollRecv() (value, ok) to PacedWire/Out; returns ok=false immediately when hasSend is false. Windowed nodes poll each input per iteration instead of parking on a blocking Recv.
  • Window loop — on first input present, record t0 (wall-clock). Each iteration: if all inputs present → fire (Done all); else if now − t0 > W → clear (Done each held input, reset HasX flags + window). Use select with a short timeout between polls to avoid busy-spin.
  • Clear semantics — call Done on each held input's In port (drains the upstream wire so a consumeGated source's WaitConsumed returns). No Fire(). Emit a window-clear trace breadcrumb for observability.
  • Clock units — real elapsed wall-clock time measured at the node (the gap between NotifyDelivered-driven arrivals). Default W to a small multiple of current simLatencyMs (~2500–3300 ms per wire).
