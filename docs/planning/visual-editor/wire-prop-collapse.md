# Wire-Prop Collapse

## Frame

Adding a named wire prop today touches 5–6 files. Goal: collapse to 1 (a
Go edit) by generating wire metadata from a Go source, mirroring the
port-source collapse that reduced port-handle scatter to a single SPEC.md
entry per kind.

## Current tax (per new wire prop)

| File | What changes | Example line |
|---|---|---|
| `nodes/Wiring/loader.go` | Add field to `specEdge` struct | `specEdge.Label string` |
| `tools/topology-vscode/src/schema/types-graph.ts` | Add field to `Edge` type | `label?: string` (line 48) |
| `tools/topology-vscode/src/schema/parse-nodes-edges.ts` | Add `opt(o.label, ...)` call to `parseEdge` | line 87 |
| `tools/topology-vscode/src/webview/rf/types.ts` | Add field to `EdgeData` interface | `label?: string` (line 52) |
| `tools/topology-vscode/src/webview/rf/adapter/spec-to-flow.ts` | Thread prop into `data:{}` block | line 163 |
| `tools/topology-vscode/src/webview/rf/adapter/flow-to-spec.ts` | Read prop back out of `d` | line 73 |
| `tools/topology-vscode/src/webview/rf/edges/SubstrateEdge.tsx` | Consume the prop in render | `displayLabel` (line 34) |

Known props requiring this full path: **`label`** (channels identity → display
label), **`valueLabel`** (transient value overlay), **`lane`** (routing
hint), **`arrowStyle`**, **`concurrent`**, **`kind`**.

`seed` is NOT a named prop — it rides inside the opaque `edge.data` blob
(`edgeData` in RF), handled by the node-data-types generated path.

## Proposed source of truth

**`wire:` struct tags on `nodes/Wiring/loader.go`'s `specEdge` struct.**

Rationale: mirrors the `wire:"data.*"` tag pattern already used for node
data fields and already parsed by `gen-node-defs`. No new Go file or
struct; the tag co-locates prop metadata with the loader's field
declaration — the one place that already owns "what an edge carries at the
Go level."

```go
type specEdge struct {
    Label        string `json:"label"        wire:"prop,optional,tsType:string"`
    ValueLabel   string `json:"valueLabel"   wire:"prop,optional,tsType:string"`
    Lane         int    `json:"lane"         wire:"prop,optional,tsType:number"`
    ArrowStyle   string `json:"arrowStyle"   wire:"prop,optional,tsType:ArrowStyle"`
    Concurrent   *bool  `json:"concurrent"   wire:"prop,optional,tsType:boolean"`
    Kind         string `json:"kind"         wire:"prop,required,tsType:EdgeKind"`
    // ...
}
```

## Generator extension

Extend `tools/gen-node-defs` (or a sibling `gen-wire-defs` if cleaner) to
parse `wire:"prop,..."` tags on `specEdge` and emit
`src/schema/wire-defs.ts`:

```ts
// GENERATED by tools/gen-node-defs — do not edit.
export interface WirePropDef {
  tsType: string;
  required: boolean;
}
export const WIRE_PROPS: Record<string, WirePropDef> = {
  label:       { tsType: "string",     required: false },
  valueLabel:  { tsType: "string",     required: false },
  lane:        { tsType: "number",     required: false },
  arrowStyle:  { tsType: "ArrowStyle", required: false },
  concurrent:  { tsType: "boolean",    required: false },
  kind:        { tsType: "EdgeKind",   required: true  },
};
```

`EdgeData` in `rf/types.ts` and the `Edge` type in `types-graph.ts` are
then derived from (or checked against) `WIRE_PROPS` rather than written
by hand.

## Migration

1. **Add Go source.** ✅ Add `wire:"prop,..."` tags to `specEdge` fields in
   `loader.go`.
2. **Extend generator.** ✅ Parse the new tags in `gen-node-defs/main.go`; emit
   `wire-defs.ts`.
3. **Replace hand-written types.** ✅ Extended generator to emit `WireProps`
   type from registry; `Edge` in `types-graph.ts` is now `WireProps & {...}`
   and `EdgeData` in `rf/types.ts` extends `WireProps`. Hand-written
   per-field declarations deleted.
4. **Replace adapter string-naming.** ✅ Made `spec-to-flow.ts` and
   `flow-to-spec.ts` iterate `WIRE_PROPS` to thread props, deleting the
   per-prop explicit chains. `kind` kept explicit (structurally required,
   drives `KIND_COLORS`).
5. **Delete duplicated schema validator entries.** ✅ `parse-nodes-edges.ts`
   `parseEdge` now loops over `WIRE_PROPS` for scalar props (`label`,
   `valueLabel`, `lane`, `concurrent`). `kind` (required enum) and
   `arrowStyle` (optional enum) remain explicit — no `ARROW_STYLES` constant
   exists and enum dispatch in the loop would add complexity for one prop.

## Verification

- `go test ./...` — loader round-trip.
- `npm run build` — TS compiles with generated types.
- `npm run check:loc` — no file over 200 LOC after changes.
- Editor smoke test: drag a wire, set `label` and `lane`, save, reload —
  both fields persist in the loaded topology.

## Out of scope

Substrate model changes; new wire props themselves; runtime semantics;
`edgeData` blob (already handled by node-data-types pattern).

## Status

**Complete.** All 5 migration steps done. Final wire-prop tax: add a new
scalar wire prop by editing one file — `nodes/Wiring/loader.go` (add the
struct field with `wire:"prop,..."` tag); the generator rebuilds
`wire-defs.ts`, and all TS consumers (types, adapters, parser) pick it up
automatically. Enum-typed props (`kind`, `arrowStyle`) still require a
one-time explicit case in `parseEdge`, but no other file changes.
