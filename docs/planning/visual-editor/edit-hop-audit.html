The edit round-trip — what can be minimized

A geometry edit makes 12 hops: webview → host → Go → host → webview. The key fact: every hop is exactly one of three kinds, and only one kind is removable. The split is 6 fixed work + 4 fixed crossings + 2 removable glue.

The three kinds of hop

6
Fixed

Does work

Hops that actually do something: compute a position, throttle the drag, parse bytes into a struct, mutate the graph, recompute geometry, draw.

This is why the path exists. Remove one and behavior changes — not just hop count. Not on the table.
4
Fixed

Process crossings

The path spans 3 processes, so it crosses 2 boundaries — each twice (out + back): webview↔host, host↔Go.

This is where it must cross. Irreducible unless you merge processes (you can't — Go owns the clock).
2
Removable

Redundant glue

Hops that re-validate a payload trusted code already produced — the host re-checks the webview's EditMsg; the webview re-checks the host's TraceEvent.

This is the entire minimization budget. Nothing else can shrink. Every saved hop comes from here.

There is one number, not two: the floor is 10. Of the 12 hops, exactly 2 are removable — both are redundant re-validations. The other 10 either do work (6) or cross a process boundary (4), and those are why and where the path exists. The only question for any minimization idea is: does it cut hop 4 or hop 11? If it touches anything green or amber, it isn't minimizing the path — it's changing the model.

The two removable hops:

  • Hop 4 — host re-validate. The host re-parses the EditMsg against the discriminated union (handle-message.ts:25) only to forward it to Go. The webview already produced it typed; nothing untrusted touched it. Drop the re-validate, pass straight to writeStdin. −1
  • Hop 11 — webview re-validate. The webview re-validates the TraceEvent (main.tsx:78) that the host already parsed at hop 9. Two validations on one payload that never left trusted code. Drop the second. −1
  • Why nothing else qualifies. Earlier this audit miscounted hops 2 and 6 as glue. They aren't: hop 2 throttles drags to one send per frame (load-shedding), and hop 6 is the only place wire bytes become a Go struct. Removing either changes behavior. They are work, so the floor is 10 — there is no lower "8".

Bridge contract (CLAUDE.md / MODEL.md): the 2 crossings stay fire-and-forget — these cuts are intra-process call-chain shortening, never new bridge messages, never an await on the TS→Go send. And never fold Go's geometry/timing (hops 7–8) up into the host to "save a crossing": that's the drift the model forbids. The crossings are load-bearing because Go owns the one clock.
