LayoutHolder — one owner, no lock

One instance per node, embedded in every node kind's struct (layout_holder.go:82). It used to carry a sync.Mutex (mu); that lock was removed (task/mutex-layout-holder) once every non-test caller was confirmed to run on the holder's own node's goroutine — the only concurrent reader anywhere was a test polling live state it did not own. The test was the one that had to change; the production model already had no second party.

0 · what a LayoutHolder actually is

Before the lock: this is one node's record of where its neighbours are, in polar terms, quantized to integer steps. Real values below are node 9's, read from topology/nodes/9/local-polars.json.

1 · what the struct holds now — two fields, six plain methods

One LayoutHolder is embedded in EVERY node kind's struct — one instance per node, not a package-wide singleton. There is no mu anymore: every accessor just reads or writes the two fields directly.

2 · what was found, and what was removed

Grepped every non-test caller of all six accessors. Every one ran on the holder's own node's goroutine. A node never reaches into a neighbour's holder — it sends a message and that neighbour's own goroutine writes its own. The only concurrent second party the race detector ever found was a test polling live state from its own goroutine while a mover goroutine wrote it. mu was deleted, not narrowed — see layout_holder.go's doc comment for the invariant that makes that safe.
